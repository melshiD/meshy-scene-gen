## Bugs Found & Fixed

### 1. Decomposed Prompts Not Stored in Job

**Problem:** When a single prompt like "A rabbi atop a hill teaches the masses" is submitted, the AI decomposes it into separate object/background prompts, but these weren't being saved to the job record - only logged to console.

**Fix Applied:**

**File: `src/lib/pipeline/job-store.ts`** - Added new function:
```typescript
export function updateJobDecomposedPrompts(
  id: string,
  objectPrompt: string,
  backgroundPrompt: string,
  mood?: string
): void {
  const job = jobs.get(id);
  if (job) {
    job.objectPrompt = objectPrompt;
    job.backgroundPrompt = backgroundPrompt;
    if (mood) {
      job.mood = mood;
    }
  }
}
```

**File: `src/lib/pipeline/generate-asset.ts`** - Added import and call:
```typescript
// After parsePrompts() in processJob():
updateJobDecomposedPrompts(jobId, objectPrompt, backgroundPrompt, mood);
```

**File: `src/types/index.ts`** - Added `mood` field to `GeneratedAsset` interface.

### 2. Storage URL Missing `/generated/` Prefix

**Problem:** Job status returned URLs like `/mesh/job-xxx/model.glb` but files are actually at `/generated/mesh/job-xxx/model.glb`.

**Root Cause:** `STORAGE_PUBLIC_URL` env var not set, and the default wasn't being applied correctly.

**Fix Applied:** Added to `.env.local`:
```
STORAGE_PUBLIC_URL=/generated
```

**Note:** The `.env.local` file needs API keys restored (they were accidentally overwritten):
```
MESHY_API_KEY=<your-key>
OPENAI_API_KEY=<your-key>
STORAGE_PUBLIC_URL=/generated
```
